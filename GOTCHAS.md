# GOTCHAS.md

Сюда записываются реальные ошибки, допущенные в процессе работы над проектом.
**Правило:** после каждой сессии (или по ходу работы) — если что-то пошло не так, добавить сюда запись. Формат: что случилось → почему → как правильно.

---

## 1. Редактирование файла после внешней модификации

**Что случилось:** попытался отредактировать `backend/package.json` сразу после `npm install`, получил ошибку `File has been modified since read`.

**Почему:** `npm install` обновил файл на диске, но у меня в контексте была старая версия. Edit tool проверяет это и блокирует запись.

**Как правильно:** всегда делать `Read` заново перед `Edit`, если между ними выполнялась команда, которая могла изменить файл (npm install, git, генераторы и т.д.).

---

## 2. Размер кнопки определяется font-size, а не padding

**Что случилось:** пользователь просил уменьшить кнопку вдвое — я несколько раз делил padding пополам, но кнопка почти не уменьшалась.

**Почему:** когда `font-size: 2rem`, именно шрифт задаёт высоту элемента. Padding `0.025rem` уже ничего не меняет — он меньше пикселя.

**Как правильно:** если нужен точный размер кнопки — задавать `width` и `height` явно + `align-items: center`. Если нужно масштабировать пропорционально — менять `font-size` и `padding` вместе.

---

## 3. Забыл добавить видимый UI авторизации

**Что случилось:** реализовал `AuthService`, обновил `CheckoutComponent`, `OrdersService` — но не добавил кнопку "Войти" в хедер и страницу `/login`. Пользователь сказал: *"на сайте не видно авторизацию"*.

**Почему:** сосредоточился на логике и данных, не прошёлся по user journey глазами пользователя.

**Как правильно:** после реализации любой фичи мысленно пройти по ней как пользователь — от точки входа (хедер, навигация) до результата. Если фича не видна без знания кода — она не готова.

---

## 4. Edit tool не срабатывает если строка не уникальна в файле

**Что случилось:** попытался заменить `font-size: 2rem` в `cart.component.scss`, получил ошибку: *"Found 2 matches... replace_all is false"*.

**Почему:** эта строка встречалась дважды в файле, и Edit tool не знает, какую именно нужно менять.

**Как правильно:** всегда включать в `old_string` достаточно окружающего контекста, чтобы строка была уникальной. Либо использовать `replace_all: true`, если нужно заменить все вхождения.
